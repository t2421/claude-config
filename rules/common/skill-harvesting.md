# スキル収穫 (Skill Harvesting)

## 原則

セッション中に生まれた**再利用可能な知見は、その場で資産化する**。
「またいつか同じことを調べ直す」を許さない。正典リポジトリは
**https://github.com/t2421/claude-config**(public)。

## 収穫の基準 — 次のいずれかに当てはまったら積極的に収穫する

1. **2つ目のプロジェクトでも使いそうな手順**が確立した
   (例: 実機配信、外部CLIへの委譲、特定APIの攻略法)
2. **ハマりどころを実体験で踏んだ**
   (エラーメッセージ → 原因 → 対処のセットは最も価値が高い)
3. 同じ説明・同じ手順を**2回目に書いていることに気づいた**
4. ユーザーが「これルール化して」「他でも使う」と言った

## 何をどこへ

| 種類 | 置き場所 |
|---|---|
| 原則・チェックリスト (what) | `rules/common/` または `rules/<言語>/` |
| 手順書・雛形・攻略法 (how) | `skills/<名前>/SKILL.md` (+ `templates/` に実行可能な雛形) |
| プロジェクト固有値 (which) | **収穫しない** — 各プロジェクトのリポジトリへ |

## ワークフロー

```bash
cd ~/projects/t2421/claude-config
# 1. rules/ か skills/ に書く (~/.claude/ を直接編集しない)
./install.sh        # 2. ~/.claude/ へ反映
git add -A && git commit && git push   # 3. 必ずpushまで
```

## 禁止事項

- **公開リポジトリである**。機密・個人識別子 (実UDID・クライアントID・氏名入り署名者名・
  トークン・内部URL) は絶対に入れない。雛形はプレースホルダ化し、コミット前に機密スキャンする
- `~/.claude/rules/` `~/.claude/skills/` の直接編集 (履歴から漏れて消失する)
- 上流ECC (`~/everything-claude-code`) への直接コミット (pullが汚れる)
